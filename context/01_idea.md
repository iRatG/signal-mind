# Идея проекта — Signal Mind / Venture Signal Engine

## Суть в одном предложении

Построить систему, которая соединяет финансовые, макроэкономические и регуляторные данные, и запускает циклического AI-агента, который сам генерирует гипотезы, проверяет их по цифрам и накапливает найденные паттерны и сигналы.

## Главная формула

```
слабый сигнал → доказательства → бизнес-возможность → проверочное действие
```

## Откуда идея

Началось как AI-инкубатор идей (сканирование arXiv, HuggingFace, GitHub).
Эволюционировало в более точную концепцию: не генерировать идеи, а **находить слабые рыночные сигналы** в данных — ранние признаки того, что в какой-то отрасли появляется боль, бюджет, регуляторное давление или технологический сдвиг.

## Что такое сигнал

Сигнал — это повторяемое наблюдение из данных, которое указывает на изменение, напряжение или возможность. Сигнал ценен если у него есть:
- Источник (откуда данные)
- Доказательство (цифры, цитаты)
- Изменение (что стало новым или усилилось)
- Покупатель (кто может заплатить за решение)

## Ключевая идея — event study подход

Одна из центральных гипотез проекта:

> Аномалии в числовых данных (резкое движение MOEX, отклонение курса, скачок зарплат) можно связать с новостями за период ДО аномалии (1 день, 1 неделя, 1 месяц).

Если паттерн "новость типа X → аномалия типа Y через N дней" повторяется стабильно — это сигнал.

Пример:
- MOEXOG резко вырос → смотрим новости за неделю до → находим "OPEC cut"
- Рубль резко упал → смотрим новости → находим "санкции на банк"
- Ключевая ставка выросла → смотрим как реагирует MOEXFN через 2 недели

## Архитектура системы по слоям

```
СЛОЙ 4: СИГНАЛЫ
  Найденные паттерны → Signal Brief → действие

СЛОЙ 3: ЦИКЛИЧЕСКИЙ АГЕНТ (Уроборос)
  Генерирует гипотезу → проверяет данные → записывает → следующая гипотеза
  Вдохновлён архитектурой Ouroboros (joi-lab) — постоянный цикл мышления

СЛОЙ 2: СВЯЗКА ДАННЫХ
  Числа → DuckDB  |  Тексты/PDF/Новости → ChromaDB (RAG)

СЛОЙ 1: ДАННЫЕ
  MOEX · Росстат · ЦБ · Ключевая ставка · Валюты · Сырьё · КГО PDF · Новости
```

## Главные принципы

- Данные первее агента — без данных агент это фантазия
- Сначала ручная проверка, потом автоматизация
- Каждый вывод должен иметь источник — не "кажется есть связь", а "вот SQL-запрос, вот числа"
- Мы не знаем заранее что найдём — задача машины искать, задача человека оценивать
- Без Red Team агент будет верить себе — нужен агент-скептик который убивает слабые сигналы

## Конечная цель (продукт)

**Signal Sprint** — за 7 дней клиент получает 3-5 доказательных сигналов в выбранной нише и один детальный launch pack (MVP, покупатели, план проверки, kill criteria).

Первая ниша: banking / audit / compliance / regulatory reporting (Россия).
