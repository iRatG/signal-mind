Зачем искать «слабые» сигналы

Финансовые рынки — странное место. Очевидные новости уже отыграны к тому моменту, когда вы успеваете прочитать заголовок. Но есть другой класс зависимостей — те, что проявляются не мгновенно, а через неделю, через месяц, через три. Рост нефти сегодня может двинуть финансовый сектор через девяносто дней — не потому что кто-то так решил, а через цепочку:

Пример лаговой цепочки
нефть ↑ → рубль укрепляется → инфляционное давление падает → ЦБ снижает ставку → банковская маржа сжимается → MOEXFN ↓
Лаг от начала до конца: ~90 дней

К тому времени, как связь становится очевидной, она уже перестаёт работать. Именно такие связи мы хотели найти систематически. Не вручную — рынок слишком большой. И не просто статистически — нужно понимать контекст, условия, режим рынка.

Так появилась идея агента, который сам генерирует гипотезы, сам их проверяет, сам учится на ошибках и несёт знания из сессии в сессию. Мы назвали его Signal Mind.

Архитектура: три петли обучения одновременно

Ключевая идея — агент должен учиться на трёх разных горизонтах одновременно. Как опытный трейдер: исправляет тактическую ошибку здесь и сейчас, развивает стратегию за сессию, и строит базу знаний на годы вперёд.

┌─────────────────────────────────────────────────────────────────┐ │ Петля 3: cross-session learning │ │ Каждая итерация → experiments.db → датасет для fine-tuning │ │ │ │ ┌──────────────────────────────────────────────────────────┐ │ │ │ Петля 2: Ouroboros — per-session │ │ │ │ гипотеза → SQL → данные → оценка → новая гипотеза │ │ │ │ │ │ │ │ ┌────────────────────────────────────────────────────┐ │ │ │ │ │ Петля 1: SQL self-repair — per-iteration │ │ │ │ │ │ ошибка → классификация (8 типов) → исправление │ │ │ │ │ └────────────────────────────────────────────────────┘ │ │ │ └──────────────────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────────────┘

Петля 1 — когда SQL не работает, агент чинит сам

DuckDB строгая. Неправильный тип в INTERVAL, неверное имя колонки, деление на ноль — всё это ошибки. Мы классифицировали восемь типов и написали repair loop: агент получает ошибку, определяет тип, пробует исправить, до трёх попыток.
Результат марафона: 22 события, 0 неудач.

Петля 2 — Ouroboros: змея, кусающая себя за хвост

Каждая итерация — полный цикл:

RAG контекст
~65 мс
Новостной поиск
~25 мс
SQL выполнение
~40 мс
LLM генерация
~8 000 мс
LLM оценка
~5 000 мс

LLM занимает 99% времени. Вся аналитическая инфраструктура — меньше 1%.

Результат предыдущей итерации влияет на следующую. Нашли что-то интересное в нефтяном секторе — следующий шаг глубже туда. Тема исчерпана — RANDOM_JUMP_PROB=0.25 выбрасывает агента в случайную из 22 тем. Это защита от зависания в локальных минимумах.

Петля 3 — строим датасет для fine-tuning

Каждая итерация пишет строку в SQLite experiments.db: гипотеза, весь контекст, сгенерированный SQL, результат, оценка. После марафона: 1 953 строки. Три датасета в одном — text-to-SQL, SQL-repair и reward model.

Ошибка, которую мы нашли сами

Когда марафон завершился и мы увидели 67.7% confirmed — это казалось невероятно высоким. Почти семь из десяти гипотез подтверждены? Мы запустили независимую верификацию: взяли топ-сигналы и вручную проверили каждый SQL на живой базе.

И нашли системный баг.

Что делал агент

Агент должен был брать данные по FTSE_CHINA_50 из правильной таблицы:

-- Правильно:
FROM market_data WHERE instrument = 'FTSE_CHINA_50'

Вместо этого он писал:

-- Что агент реально делал:
SELECT imoex_close AS ftse_china_50   -- ← подмена колонки
FROM v_market_context
Суть ошибки
Агент брал imoex_close из аналитического вью и переименовывал колонку. Это означало: он мерил корреляцию IMOEX с MOEXOG — двух российских индексов. Конечно, они коррелируют (r ≈ 0.88). Это не сигнал. Это тавтология. Та же ошибка повторялась для DXY, MSCI_INDIA, DJ_SOUTH_AFRICA.

Масштаб

Около 60% «confirmed» сигналов с максимальным score были артефактами. Реальный confirmed rate — 15–25%, не 67.7%. Четыре из девяти кластеров топ-сигналов — иллюзия.

Что мы сделали

Не удалили. Не скрыли. Задокументировали:

  • Паттерн #8 в db/forbidden_patterns.md — «instrument aliasing»
  • Явные предупреждения в schema.py с верифицированными числами
  • Принцип «Data source integrity» в analysis_principles.md
  • Aliasing-примеры в experiments.db помечены как отрицательные для Dataset B
Найти собственную ошибку труднее, чем найти сигнал. Но именно это делает результат настоящим. Теперь experiments.db содержит и правильные, и неправильные примеры — а это ценнее, чем датасет только с идеальными ответами.

Индекс доллара не предсказывает рубль. USD/RUB полностью отвязан от DXY из-за санкций и валютного контроля. Классический FX-индикатор, который работает везде в мире — на российском рынке не работает. Это само по себе важное знание.

Лучший сигнал: нефть предсказывает банки в минус через три месяца

Самый нетривиальный результат — обратная корреляция Brent с MOEXFN с лагом 90 дней. r = −0.702 на 799 торговых днях.

На первый взгляд звучит странно: нефть растёт — а финансовый сектор падает? Но логика есть:

Цепочка механизма
Brent ↑ → рубль укрепляется (нефтяная выручка в страну)
→ инфляционное давление падает
→ ЦБ получает возможность снизить ставку
→ банковская маржа сжимается
→ MOEXFN ↓

Или: крепкий рубль снижает рублёвые доходы нефтяных заёмщиков → кредитный портфель банков под давлением

Верифицированный SQL:

WITH t AS (
    SELECT
        d.trade_date,
        d.close           AS brent_price,
        s.moexfn_finance
    FROM market_data d
    JOIN v_moex_sectors s
        ON s.trade_date = d.trade_date + INTERVAL 90 DAYS
    WHERE d.instrument = 'BRENT'
      AND d.trade_date BETWEEN '2022-01-01' AND '2025-12-31'
      AND d.close IS NOT NULL
      AND s.moexfn_finance IS NOT NULL
)
SELECT
    ROUND(CORR(brent_price, moexfn_finance), 4) AS correlation,
    COUNT(*)                                      AS n_observations
FROM t;

-- Результат: correlation = -0.7023, n = 799

На 799 торговых днях, четыре года данных, стабильная обратная корреляция. Это не шум — это структурная особенность российской экономики.

Открытие внутри открытия: сигналы меняют знак

Может быть, главная аналитическая находка проекта — не конкретные корреляции, а то, что сигналы режимозависимы. Один и тот же фактор работает по-разному в разных состояниях рынка:

СигналУсловиеrn
MSCI_INDIA → MOEXFN Сильный рубль (USD/RUB < 80) +0.77 223
Слабый рубль (USD/RUB > 80) +0.16 374
Banking news → MOEXFN Ставка ЦБ < 15% +0.549 383
Ставка ЦБ ≥ 15% (сейчас) +0.010 355

Правильный вопрос не «работает ли этот сигнал», а «в каком режиме рынка он работает». Рынок — не одна система с постоянными параметрами. Это набор режимов, переключающихся под влиянием ставки, санкционного давления, курса рубля.
