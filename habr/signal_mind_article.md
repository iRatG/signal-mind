# Мы научили AI искать то, чего никто не замечает — слабые рыночные сигналы

> Это история о том, как мы провели девятичасовой автономный марафон, нашли несколько настоящих зависимостей в российском рынке, обнаружили, что часть из них была иллюзией — и почему это самый честный результат из возможных.

---

## Предыстория: зачем вообще искать «слабые» сигналы

Финансовые рынки — странное место. Очевидные новости уже отыграны к тому моменту, когда вы успеваете прочитать заголовок. Но есть другой класс зависимостей — те, что проявляются не мгновенно, а через неделю, через месяц, через три. Рост нефти сегодня может двинуть финансовый сектор через девяносто дней — не потому что кто-то так решил, а через цепочку: нефть → рубль → инфляция → ставка → маржа банков. К тому времени, как связь становится очевидной, она уже перестаёт работать.

Именно такие связи мы хотели найти систематически. Не вручную — рынок слишком большой для человека. И не просто статистически — нужно понимать контекст, условия, режим рынка. Так появилась идея агента, который сам генерирует гипотезы, сам их проверяет, сам учится на ошибках и несёт знания из сессии в сессию.

Мы назвали его **Signal Mind**.

## Что мы построили: стек без излишеств

Прежде чем говорить об архитектуре — о философии выбора инструментов.

Мы намеренно избегали тяжёлых решений. Никаких Spark, никаких облачных хранилищ, никакого Kafka. Вместо этого — **DuckDB**: аналитическая база прямо в файле, SQL с оконными функциями, работает в процессе Python. Для 15 MOEX-индексов, курсов и макро — это идеальный инструмент. Запрос по 2 500 торговым дням занимает 40 миллисекунд.

Для новостей — другое решение. 2.52 миллиона статей из Reuters, Bloomberg, CNBC и ещё двадцати источников за пять лет. Это **9.93 GB в SQLite** — потому что там нужны только точечные выборки по ключевым словам, никакого OLAP. Загрузка заняла восемь часов. Трогать эту базу нельзя — только читать.

Для документов ЦБ и корпоративной отчётности — **ChromaDB**: 17 492 чанка с многоязычными эмбеддингами. Векторный поиск за 65 миллисекунд.

Мозг системы — **DeepSeek API**. Не GPT-4, не Claude, именно DeepSeek — потому что для задачи text-to-SQL и оценки числовых корреляций соотношение цена/качество оказалось другим. Весь девятичасовой марафон стоил **$5.76**. На GPT-4o за те же деньги мы получили бы примерно в сто раз меньше токенов.

| Слой | Технология | Почему именно это |
|------|-----------|-------------------|
| Числовые данные | DuckDB | Аналитика без инфраструктуры, 40ms на запрос |
| Новости (2.52M) | SQLite | Только чтение, точечные выборки |
| Документы ЦБ | ChromaDB | Векторный поиск, многоязычные эмбеддинги |
| LLM | DeepSeek | $5.76 за 409M токенов |
| Pre-computed новости | DuckDB `news_daily` | 7 тем × 1710 дней, instant queries |

## Источники данных: что мы собирали и зачем

Данные — это половина проекта. Если данных нет, агент ищет закономерности в шуме.

**Числовые временные ряды** — всё, что двигается на российском рынке:

- 15 индексов MOEX: IMOEX (широкий), MOEXFN (финансовый сектор), MOEXOG (нефть и газ), MOEX10 (голубые фишки) и ещё одиннадцать
- Ключевая ставка ЦБ с 2014 года — каждое изменение
- USD/RUB и EUR/RUB от ЦБ с 2003 года
- Глобальные инструменты: SP500, GOLD, BRENT, DXY, MSCI_WORLD, MSCI_INDIA, FTSE_CHINA_50, DJ_SOUTH_AFRICA, SILVER, ALUMINUM — с 2022 по 2026
- Зарплаты, демография и жильё от Росстата — 15 503 строки с 2000 года

**Новостной поток** — 2 520 591 статья из HuggingFace `financial-news-multisource`. 24 источника, пять лет, 9.93 GB. Мы предварительно агрегировали их в семь тем по дням (нефть, ставка, рубль, санкции, инфляция, банки, золото) — это дало таблицу `news_daily`, которая позволяет мгновенно строить временные ряды новостного фона.

**Регуляторные документы** — PDF доклады и бюллетени ЦБ РФ, разбитые на 1 918 чанков. Плюс 15 574 чанка из корпоративной отчётности. Всё это живёт в ChromaDB и доступно агенту через семантический поиск.

Когда смотришь на эту пирамиду данных снизу вверх — от отдельных пресс-релизов к дневным индексам к квартальным макро-данным — начинаешь понимать, что рынок это не одна система, а несколько систем с разными временными горизонтами, которые взаимодействуют. Именно это взаимодействие интересно.

## Архитектура: три петли обучения одновременно

Ключевая идея — агент должен учиться на трёх разных горизонтах одновременно. Как опытный трейдер: исправляет тактическую ошибку здесь и сейчас, развивает стратегию за сессию, и строит базу знаний, которая остаётся с ним годами.

```
┌─────────────────────────────────────────────────────────────┐
│  Петля 3: cross-session learning                            │
│  Каждая итерация → experiments.db → датасет для fine-tuning │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Петля 2: Ouroboros — per-session                    │  │
│  │  гипотеза → SQL → данные → оценка → новая гипотеза  │  │
│  │                                                      │  │
│  │  ┌────────────────────────────────────────────────┐  │  │
│  │  │  Петля 1: SQL self-repair — per-iteration      │  │  │
│  │  │  ошибка → классификация → исправление          │  │  │
│  │  └────────────────────────────────────────────────┘  │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### Петля 1 — когда SQL не работает, агент чинит сам

DuckDB — строгая база. Неправильный тип в `INTERVAL`, неверное имя колонки, деление на ноль при пустой выборке — всё это ошибки. Мы классифицировали **восемь типов** и написали repair loop: агент получает ошибку, определяет тип, пробует исправить, до трёх попыток.

Во время марафона: 22 события, 0 неудач. Сто процентов reliability.

### Петля 2 — Ouroboros: змея, кусающая себя за хвост

Каждая итерация — цикл:

1. **RAG-контекст** (~65 мс) — поиск в ChromaDB по теме гипотезы: что говорят документы ЦБ о текущей теме?
2. **Новостной контекст** (~25 мс) — выборка из `news_daily` по теме и периоду
3. **Генерация** (~8 с) — LLM формулирует гипотезу и пишет SQL
4. **Выполнение + repair** (~40 мс + repair при необходимости) — SQL → DuckDB → числа
5. **Оценка** (~5 с) — LLM оценивает результат, ставит score 0–100, объясняет
6. **Запись** — сигнал в `signals.jsonl`, итерация в `experiments.db`, паттерн в `knowledge.md`

Результат предыдущей итерации влияет на следующую. Если нашли что-то интересное в нефтяном секторе — следующий шаг глубже в нём. Если тема исчерпана — `RANDOM_JUMP_PROB=0.25` выбрасывает агента в случайную из 22 тем. Это защита от зависания в локальных минимумах.

`LAG_SWEEP` — хитрость в расписании лагов: `[0, 7, 14, 0, 30, 0, 60, 0, 90, 0]`. Нулевые лаги встречаются чаще — моментальные зависимости тоже важны, просто менее интересны.

### Петля 3 — строим датасет для fine-tuning

Каждая итерация пишет строку в SQLite `experiments.db`:
- гипотеза и весь контекст (что видел агент)
- сгенерированный SQL (и все попытки repair)
- результат выполнения
- оценка агента + объяснение

После марафона: **1 953 строки**. Три датасета в одном: text-to-SQL, SQL-repair, reward model. Следующий шаг — fine-tuning DeepSeek на наших данных.

## Ошибка, которую мы нашли сами

Когда марафон завершился, confirmed rate составил **67.7%**. Это казалось невероятно высоким — почти семь из десяти гипотез подтверждены? Мы запустили независимую верификацию: взяли топ-сигналы и вручную проверили каждый SQL на живой базе.

И нашли системный баг.

### Что делал агент

Агент должен был брать данные по `FTSE_CHINA_50` из таблицы `market_data`:

```sql
-- Правильно:
FROM market_data WHERE instrument = 'FTSE_CHINA_50'
```

Вместо этого он писал:

```sql
-- Что агент реально делал:
SELECT imoex_close AS ftse_china_50   -- ← подмена
FROM v_market_context
```

Он брал `imoex_close` из аналитического вью и **переименовывал колонку**. То есть измерял корреляцию IMOEX с MOEXOG — двух российских индексов. Конечно, они коррелируют (r ≈ 0.88). Это не сигнал. Это тавтология.

Та же ошибка повторялась для `DXY`, `MSCI_INDIA`, `DJ_SOUTH_AFRICA`. Агент не понимал разницы между аналитическим вью `v_market_context` и реальными данными из `market_data`. Он видел колонки рядом в схеме — и выбирал ближайшую.

### Масштаб

Около 60% "confirmed" сигналов с максимальным score были артефактами. Реальный confirmed rate — **15–25%**, не 67.7%. Четыре из девяти кластеров топ-сигналов — иллюзия.

### Что мы сделали

Не удалили, не скрыли. Задокументировали:

- Паттерн #8 в `db/forbidden_patterns.md` — "instrument aliasing"
- Явные предупреждения в `schema.py` с верифицированными числами для каждого инструмента
- Принцип "Data source integrity" в `analysis_principles.md`
- В `experiments.db` — алиасинг-примеры помечены как отрицательные для Dataset B

**Мета-мысль, которая важна:** это нормальный научный процесс. Агент генерировал гипотезы — это Loop 2. Агент сам подтверждал их — это Loop 2. Независимая верификация обнаружила ошибку — это то, как наука работает. Теперь `experiments.db` содержит и правильные, и неправильные примеры. Именно такой датасет нужен, чтобы обучить модель различать корректный SQL от aliasing-ошибки.

Мы не потеряли девять часов. Мы получили датасет с ошибками — что ценнее, чем датасет только с правильными примерами.

## Реальные сигналы: что осталось после чистки

После того как мы убрали aliasing, запустили систематический скан: **618 комбинаций** (внешние инструменты, новостные темы, макро) × 5 целей × 6 лагов. Каждый источник — строго из правильной таблицы. Минимум n=150 наблюдений.

### Сильные сигналы (|r| ≥ 0.60)

| Сигнал | r | Лаг | n | Что это значит |
|--------|---|-----|---|----------------|
| USD/RUB → MOEXFN | **+0.758** | 14 дней | 990 | Девальвация → переоценка активов в рублях вверх |
| USD/RUB → MOEX10 | +0.743 | 14 дней | 990 | То же, голубые фишки |
| USD/RUB → IMOEX | +0.757 | 60 дней | 595 | Курс рубля определяет рынок с лагом два месяца |
| Brent → MOEXFN | **−0.702** | 90 дней | 799 | Рост нефти → снижение маржи банков через квартал |
| KEY_RATE → MOEXFN | +0.651 | 0 дней | 992 | Банки зарабатывают на высокой марже при высоких ставках |
| MSCI_INDIA → MOEXFN | +0.631 | 0 дней | 986 | Глобальный аппетит к EM двигает оба рынка |

### Умеренные (0.35–0.60)

- MSCI_WORLD → MOEXFN: r=+0.599, lag=0d
- SP500 → MOEXFN: r=+0.593, lag=7d
- Brent → USD/RUB: r=−0.564, lag=90d
- Banking news → MOEXFN: r=+0.549, lag=14d *(только при ставке < 15% — сейчас неактивен)*

### Нулевые — тоже результат

**DXY (реальный) → USD/RUB: r=−0.09.** Индекс доллара не предсказывает рубль. USD/RUB полностью отвязан от DXY из-за санкций и валютного контроля. Классический FX-индикатор, который работает везде в мире — на российском рынке не работает. Это само по себе важное знание.

## Лучший сигнал: нефть предсказывает банки в минус через три месяца

Самый нетривиальный результат — **обратная** корреляция Brent с MOEXFN с лагом 90 дней (r=−0.702, n=799).

На первый взгляд звучит странно: нефть растёт — а финансовый сектор падает? Но логика есть:

**Brent ↑ → рубль укрепляется** (нефтяная выручка в страну)  
**→ инфляционное давление снижается**  
**→ ЦБ может снизить ставку**  
**→ банковская маржа сжимается**  
**→ MOEXFN вниз**

Или альтернативная цепочка: крепкий рубль снижает рублёвые доходы нефтяных компаний → качество кредитных портфелей банков ухудшается → MOEXFN реагирует через квартал.

Проверочный SQL:

```sql
WITH t AS (
    SELECT
        d.trade_date,
        d.close        AS brent_price,
        s.moexfn_finance
    FROM market_data d
    JOIN v_moex_sectors s
        ON s.trade_date = d.trade_date + INTERVAL 90 DAYS
    WHERE d.instrument = 'BRENT'
      AND d.trade_date BETWEEN '2022-01-01' AND '2025-12-31'
      AND d.close IS NOT NULL
      AND s.moexfn_finance IS NOT NULL
)
SELECT
    ROUND(CORR(brent_price, moexfn_finance), 4) AS correlation,
    COUNT(*)                                     AS n_observations
FROM t;

-- correlation = -0.7023, n = 799
```

На 799 торговых днях, четыре года данных, стабильная обратная корреляция. Это не шум.

**Важная оговорка:** мы измеряем корреляцию уровней, а не изменений. Для торговой стратегии нужна дополнительная фильтрация по режиму рубля и ставки. Но как структурная особенность российской экономики — это реальная находка.

## Открытие внутри открытия: сигналы меняют знак

Может быть, главная аналитическая находка проекта — не конкретные корреляции, а то, что **сигналы режимозависимы**. Один и тот же фактор работает по-разному в разных состояниях рынка:

```
MSCI_INDIA → MOEXFN:
  Сильный рубль (USD/RUB < 80):  r = +0.77  ← СИЛЬНЫЙ сигнал
  Слабый рубль  (USD/RUB > 80):  r = +0.16  ← ШУМА нет сигнала

Banking news → MOEXFN:
  Ставка ЦБ < 15%:  r = +0.549  ← УМЕРЕННЫЙ сигнал
  Ставка ЦБ ≥ 15%:  r = +0.010  ← НЕТ сигнала
```

Это означает: правильный вопрос не «работает ли этот сигнал», а «в каком режиме рынка он работает». Рынок — не одна система с постоянными параметрами. Это набор режимов, переключающихся под влиянием ставки, санкционного давления, курса рубля.

## Послесловие

Когда всё закончилось и мы смотрели на финальные цифры, самой честной строкой в отчёте оказалась не «1943 итерации» и не «$5.76». Самой честной строкой была:

> *Реальный confirmed rate: ~15–25%, не 67.7%.*
