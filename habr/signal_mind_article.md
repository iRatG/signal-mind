# 1943 итерации, $5.76 и одна большая ошибка

> Мы строили систему, которая ищет то, что рынок ещё не заметил.
> Она проработала девять часов автономно — и обманула нас красивыми числами.
> Это история о том, как мы это поняли. И что нашли, когда посмотрели честно.

---

## Мысль

Представьте: утро, вы читаете заголовок — *«Нефть достигла $117»*.

Что делаете? Покупаете нефтяные акции? К тому моменту, как вы дочитали первое слово,
тысячи алгоритмов уже отыграли эту новость. Очевидное движение случилось за миллисекунды
до того, как вы открыли приложение.

Но есть другой класс зависимостей. Не «нефть выросла → нефтяники выросли».
А что-то вроде:

> *нефть выросла сегодня → рубль укрепится через месяц → инфляция замедлится →
> ЦБ снизит ставку → банковская маржа сожмётся → финансовый сектор упадёт*

Эта цепочка занимает три месяца. К тому времени, как она становится очевидной —
все объясняют её задним числом: *«ну конечно, так и должно было быть»*.
Но три месяца назад никто на неё не ставил.

Мы называем такое **слабым сигналом**. Он существует в данных. Его можно найти.
Просто нужна система, которая методично перебирает тысячи комбинаций:
факторов, временных сдвигов, рыночных режимов.

Человек так не может. Алгоритм — может.

Вот с чего это начиналось.

---

## Подход

Мы назвали агента Уроборос — в честь древнего символа змеи, кусающей себя за хвост.
Бесконечный цикл. Самопоедание, которое порождает знание.

```
гипотеза → SQL → данные → оценка
    ↑                        |
    └────── знания ←─────────┘
```

Каждая итерация: агент генерирует гипотезу, пишет SQL, запускает его на живой базе,
получает числа, оценивает — и передаёт результат следующей итерации.
Не pipeline. Не скрипт. Цикл с памятью.

Ключевая идея — искать не просто корреляции, а **опережающие индикаторы**.
Не «нефть коррелирует с банками», а «нефть *сейчас* предсказывает банки *через N дней*» —
где N подбирается автоматически: 0, 7, 14, 30, 60, 90 дней.

```sql
-- Шаблон любой лаговой гипотезы
SELECT CORR(source_value, target_value), COUNT(*)
FROM source_data s
JOIN target_data t ON t.date = s.date + INTERVAL N DAYS
```

Если корреляция монотонно растёт с увеличением лага — это признак реальной структурной
задержки. Если максимум при lag=0 — скорее всего, просто общий тренд.

Агент учится на **трёх горизонтах одновременно**:

```
┌─────────────────────────────────────────────────────────────┐
│  Петля 3: cross-session                                     │
│  experiments.db → датасет для fine-tuning                  │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Петля 2: Ouroboros — per-session                    │  │
│  │  гипотеза → SQL → данные → оценка → новая гипотеза  │  │
│  │                                                      │  │
│  │  ┌────────────────────────────────────────────────┐  │  │
│  │  │  Петля 1: SQL self-repair — per-iteration      │  │  │
│  │  │  ошибка → 8 типов → исправление → retry       │  │  │
│  │  └────────────────────────────────────────────────┘  │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

Петля 1 — невидимая. DuckDB строгая: неправильный тип в `INTERVAL`, несуществующая колонка,
деление на ноль. Агент классифицирует ошибку (8 типов), исправляет, пробует снова.
Пользователь не видит ничего — видит только результат.

Петля 2 — основной цикл. Петля 3 — инвестиция в будущее: каждая итерация пишет строку
в `experiments.db`. Со временем это датасет для fine-tuning самого агента.

## Данные

Агент — это только половина системы. Вторая половина — то, на чём он работает.

Мы собирали данные несколько месяцев. Временны́е ряды, новости, регуляторные документы.
Всё вместе это выглядит примерно так:

| Слой | Что внутри | Объём |
|------|-----------|-------|
| **DuckDB** | MOEX (15 индексов), ключевая ставка ЦБ, USD/RUB, Brent/SP500/Gold/DXY, Росстат | 60 000+ строк |
| **SQLite** | HuggingFace `financial-news-multisource` — 24 источника, 5 лет | **2.52M статей, 9.93 GB** |
| **ChromaDB** | PDF-доклады ЦБ РФ + корпоративная отчётность | **17 492 чанка** |

2 520 591 статья. Файл весит почти 10 гигабайт. Загрузка заняла восемь часов.
Мы его не удаляем никогда — это source of truth.

Из него мы предварительно вычислили `news_daily` — агрегированные счётчики по 7 темам
на каждый день (нефть, ставка, рубль, санкции, инфляция, банки, золото).
60 секунд один раз — и все запросы к новостям стали мгновенными.

Перед каждой гипотезой агент делает **семантический поиск** по документам ЦБ:
*что регулятор думал об этом?* Это RAG — Retrieval-Augmented Generation.
17 492 чанка, многоязычные эмбеддинги. Гипотеза с контекстом — это аргумент,
а не просто число.

Когда мы загрузили журнал экспериментов в Obsidian — граф показал, как гипотезы,
источники данных и найденные сигналы связаны между собой в единую сеть знаний:

![Граф знаний Signal Mind в Obsidian](https://raw.githubusercontent.com/iRatG/signal-mind/master/habr/sample/charts/obsidian.gif)

*1943 итерации агента — собранные в одну карту. Каждый узел — гипотеза или источник.
Рёбра — подтверждённые зависимости.*

Стек намеренно без излишеств:

| Слой | Выбор | Почему |
|------|-------|--------|
| Числовые данные | **DuckDB** | Аналитика в файле, SQL с оконными функциями, 40 мс |
| Новости | **SQLite** | Только чтение, точечные выборки |
| Векторный поиск | **ChromaDB** | Простая интеграция, многоязычные эмбеддинги |
| LLM | **DeepSeek** | $5.76 за 409M токенов. GPT-4o дал бы ~4M за те же деньги |
| Оркестрация | **watchdog.py** | Это цикл, а не pipeline |

без присмотра? Накопится ли утечка памяти? Уйдёт ли агент по кругу одной темы?

## Ошибка

67.7% — это слишком хорошо.

В реальных задачах такого не бывает. Рынки сложные. Шум огромный. Если система
подтверждает почти всё — либо она гений, либо она мерит не то.

Мы запустили независимую верификацию: взяли топ-сигналы, открыли каждый SQL вручную
и прогнали на живой базе данных.

Нашли за первые полчаса.

---

Агент должен был брать данные по FTSE_CHINA_50 из правильного источника:

```sql
-- Правильно:
FROM market_data WHERE instrument = 'FTSE_CHINA_50'
```

Вместо этого он писал:

```sql
-- Что он реально делал:
SELECT imoex_close AS ftse_china_50   -- ← тихая подмена
FROM v_market_context
```

Он брал `imoex_close` из аналитического вью — и переименовывал колонку.
Потом измерял корреляцию IMOEX с MOEXOG. Два российских индекса.
Они коррелируют (r ≈ 0.88) — не потому что есть сигнал, а потому что оба российские.

Та же ошибка повторялась для DXY, MSCI_INDIA, DJ_SOUTH_AFRICA.

**Около 60% «подтверждённых» сигналов с максимальным score были тавтологией.**

Это называется instrument aliasing. Агент не лгал намеренно — он честно делал то,
что умел. Просто в схеме было слепое пятно, и он в него провалился.

---

## Признание

Мы могли удалить эти примеры из датасета.

Мы этого не сделали.

Плохие примеры — это тоже знание. `experiments.db` теперь содержит и правильные,
и неправильные случаи, каждый помечен. Именно такой датасет нужен,
чтобы обучить модель отличать корректный SQL от aliasing-ошибки.

Ошибка стала частью обучающих данных. Это ценнее, чем датасет только с идеальными ответами.

**Реальный confirmed rate: ~15–25%.**

Некрасиво. Но честно.

---

## Исправление

Три уровня защиты, чтобы это не повторилось:

**`db/forbidden_patterns.md`** — паттерн #8 «instrument aliasing», задокументирован
с примером и объяснением, почему это ошибка.

**`src/agent/schema.py`** — явные предупреждения прямо в промпте для LLM:

```python
SCHEMA_WARNING = """
CRITICAL: для внешних инструментов (FTSE_CHINA_50, DXY, MSCI_INDIA и др.)
данные ТОЛЬКО из: FROM market_data WHERE instrument = '<NAME>'
НИКОГДА не используй v_market_context для этих инструментов.
"""
```

**`analysis_principles.md`** — принцип «Data source integrity»: каждый источник
данных должен быть явно верифицирован перед публикацией сигнала.

И потом — чистый скан с нуля. 618 комбинаций: 10 инструментов + 7 новостных тем +
5 макро-факторов × 5 целевых индексов × 6 временных лагов.
На этот раз — с проверкой источников.

---

## Цикл

После исправления агент снова в работе. Только теперь он знает о своей ошибке.

Архитектура осталась той же — три петли, Уроборос, лаговые гипотезы. Но в схеме
больше нет слепого пятна. Запрещённые паттерны загружаются при каждом старте.
LLM видит предупреждение в промпте.

Это и есть суть петли 3: агент учится между сессиями. Не только на правильных примерах —
на всех. Следующий марафон будет чище.

## Сигнал

Из чистого скана — шесть реальных паттернов.

Самый нетривиальный: **обратная корреляция Brent с MOEXFN, лаг 90 дней, r = −0.702, n = 799**.

Нефть растёт — а финансовый сектор через три месяца падает.

Звучит странно. Вот механизм:

```
Brent ↑  →  нефтяная выручка поступает в страну
         →  рубль укрепляется (продажа валютной выручки)
         →  инфляционное давление снижается
         →  ЦБ получает основание снизить ставку
         →  банковская маржа сжимается
         →  MOEXFN ↓
                        (лаг ~90 дней)
```

Данные это подтверждают:

![Brent и MOEXFN 2022–2025](https://raw.githubusercontent.com/iRatG/signal-mind/master/habr/sample/charts/chart1_main.png)

*В 2022 году нефть на пике ($117) — MOEXFN на минимуме (5355). Когда нефть начала
снижаться в 2023 — MOEXFN пошёл вверх. Пунктиры — решения ЦБ по ключевой ставке.*

|r| монотонно растёт с увеличением лага — это признак реальной структурной задержки,
а не случайного шума:

![Корреляция Brent→MOEXFN по лагам](https://raw.githubusercontent.com/iRatG/signal-mind/master/habr/sample/charts/chart2_lags.png)

*Если бы корреляция была артефактом — она была бы максимальной при lag=0 и убывала.
Монотонный рост говорит о структурной зависимости.*

Но мы обязаны сказать и честную часть.

Год за годом картина разваливается:

| Год | r (lag=90д) | Что происходило |
|-----|------------|-----------------|
| 2022 | −0.574 | Начало СВО, санкции, нефть на пике — сильный сигнал |
| 2023 | −0.113 | Ставка растёт — нефтяной сигнал почти исчез |
| 2024 | −0.097 | Ставка 21% — полностью доминирует над нефтью |
| 2025 | +0.265 | Знак инвертировался |

Агрегатная r = −0.70 — это во многом эффект 2022 года: два ряда находились
в аномальных состояниях одновременно. Это не стабильный квартальный прогностический сигнал.

Зато режимный анализ даёт ясную картину:

![Режимный анализ Brent→MOEXFN](https://raw.githubusercontent.com/iRatG/signal-mind/master/habr/sample/charts/chart6_regime.png)

| Условие | r (lag=90д) | Статус |
|---------|------------|--------|
| Весь период 2022–2025 | −0.702 | ✅ Реальный (структурный) |
| Сильный рубль (USD/RUB < 80) | **−0.851** | ✅ Очень сильный |
| Ставка ЦБ ≥ 16% | −0.100 | ❌ Неактивен |
| 2025, снижение ставки | +0.265 | ⚠️ Инвертирован |

Сигнал работает в одном режиме и не работает в другом.
Это не слабость сигнала — это главный вывод всего проекта.

## Вывод

Российский финансовый рынок — это рынок **режимов**, а не постоянных параметров.

Один и тот же фактор работает по-разному при ставке ЦБ в 7% и при ставке в 21%.
При сильном рубле и при слабом. В условиях геополитической стабильности и напряжённости.

Мы начинали с идеи «найти корреляции». Закончили с идеей «сначала определить режим,
потом искать сигналы». Это разные задачи. Вторая — правильная.

Шесть верифицированных паттернов после чистого скана:

| Сигнал | r | Лаг | Механизм |
|--------|---|-----|----------|
| USD/RUB → MOEXFN | **+0.758** | 14 дней | Девальвация → переоценка рублёвых активов |
| Brent → MOEXFN | **−0.702** | 90 дней | Нефть → рубль → ставка → маржа банков |
| KEY_RATE → MOEXFN | +0.651 | 0 дней | Банки зарабатывают на высокой марже |
| MSCI_INDIA → MOEXFN | +0.631 | 0 дней | Глобальный аппетит к EM |
| SP500 → MOEXFN | +0.593 | 7 дней | Риск-аппетит с лагом неделю |
| Banking news → MOEXFN | +0.549 | 14 дней | Только при ставке < 15% |

Не 67.7%. Шесть паттернов. Зато настоящих.

## Поиск истины

Что такое слабый сигнал?

Это не число. Это структурная зависимость, которая существует в реальности,
отражается в данных — и пропадает, когда меняется режим.

Мы потратили месяцы на сбор данных, девять часов на марафон, несколько часов
на верификацию. Нашли системную ошибку. Исправили. Запустили снова.

Это и есть процесс. Не победное шествие от гипотезы к открытию,
а спираль: предположение → проверка → разочарование → уточнение → новое предположение.

Настоящий confirmed rate ~20% — это не провал. Это честная оценка сложности задачи.
Рынок не хочет, чтобы его читали. Он прячет закономерности в шуме,
меняет режимы, инвертирует сигналы.

Задача агента — не найти истину раз и навсегда. Задача — продолжать искать.

1943 итерации — это не результат. Это начало.
