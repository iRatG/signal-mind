## 1. Суть идеи в одном абзаце

Автономная AI-система непрерывно сканирует научные источники (arXiv, HuggingFace, GitHub, Hacker News и др.), обнаруживает потенциально прорывные идеи, обдумывает их, скрещивает концепции между собой и доводит лучшие до состояния «готового бизнес-зерна»: описание идеи, целевая аудитория, предварительные цифры рынка, MVP-лендинг. Всё это публикуется на платформе с подпиской. Сообщество голосует, обсуждает, докручивает. Результат — **работающий бизнес-инкубатор, где идеи генерирует и оформляет AI, а люди выбирают лучшее и реализуют.**

---

## 2. Откуда появилась идея

Разговор шёл по такой цепочке:

1. Начали с парсинга датасетов с HuggingFace (топ-100 трендинговых)
2. Рассмотрели проект **Ouroboros** (joi-lab) — самомодифицирующийся AI-агент с consciousness loop, Telegram-управлением, memory persistence
3. Добавили **arXiv** как источник научных статей → Python-скрипт для автоматической генерации Obsidian-заметок
4. Добавили **HuggingFace** как второй источник данных и датасетов
5. Обсудили возможность использовать Ouroboros как **постоянный циклический агент** (не разовый запуск, а живущий процесс)
6. Пришли к концепции: такой агент может не просто собирать информацию, но **находить идеи, обдумывать их, ранжировать**
7. Финальный шаг: превратить это в **коммерческую платформу** — бизнес-инкубатор с подпиской

## 3. Как это работает — полная схема

### 3.1 Слой сбора данных (Collector Agents)

Постоянно работающие фоновые агенты, каждый отвечает за свой источник:

- **arXiv Agent** — парсит новые статьи по категориям `cs.AI`, `cs.LG`, `econ`, `q-fin`, поиск по ключевым словам
- **HuggingFace Agent** — мониторит новые датасеты и модели, особенно с неожиданными применениями
- **GitHub Trending Agent** — отслеживает репозитории с быстрым ростом звёзд
- **Hacker News Agent** — мониторит Show HN, Ask HN, ветки про новые технологии
- **Patent Agent** (опционально) — USPTO, EPO — свежие патенты как сигнал технологических трендов

Все данные → **Vector Store** (ChromaDB или Qdrant, локально) в виде эмбеддингов.

### 3.2 Слой синтеза (Synthesis Agent)

Периодически (раз в несколько часов) читает новые поступления из Vector Store и:

- Ищет **семантические пересечения** между концепциями из разных источников
- Генерирует **«insight-заметки»**: «технология X из статьи arXiv + проблема Y из HN + датасет Z с HuggingFace = потенциальный продукт»
- Задаёт самому себе вопросы: «кто будет платить за это?», «какая боль решается?», «есть ли аналоги?»

### 3.3 Слой consciousness (Ouroboros-based)

Вдохновлён архитектурой `consciousness.py` из Ouroboros:

- Постоянный фоновый цикл мышления
- Возвращается к старым идеям с новым контекстом
- Может инициировать дополнительный поиск по интересной теме
- Ведёт внутренний «дневник мышления» в Obsidian vault

### 3.4 Слой оценки (Ranking Agent)

Перед публикацией каждая идея проходит оценку по критериям:

| Критерий | Вес | Описание |
|---|---|---|
| Новизна | 25% | Насколько идея отличается от существующих решений |
| Реализуемость | 25% | Можно ли сделать MVP за разумные деньги/время |
| Размер рынка | 20% | Есть ли достаточно большая аудитория |
| Технологическая готовность | 15% | Существуют ли нужные инструменты прямо сейчас |
| Релевантность тренду | 15% | Совпадает ли с направлением рынка |

Используется **multi-model review**: несколько LLM оценивают независимо, результат усредняется.

### 3.5 Слой генерации контента (Content Agent)

Для каждой идеи, прошедшей порог рейтинга, автоматически генерируется:

- **Описание идеи** (проблема → решение → целевая аудитория → дифференциация)
- **Базовые финансовые метрики** (TAM/SAM/SOM оценка, предполагаемая Unit Economics)
- **MVP-лендинг** (однострочный HTML, готовый к размещению)
- **Список из 10 первых шагов** для реализации
- **Риски и возражения** (честная оценка слабых мест)

### 3.6 Платформа (Frontend)

Сайт с подпиской, где подписчики видят поток идей:

- Карточка идеи с кратким описанием
- Голосование (лайк/диз лайк, сохранить в избранное)
- Комментарии — люди докручивают идею
- Теги и фильтры по нишам
- Лендинг каждой идеи как отдельная страница

## 4. Бизнес-модель

### Уровни подписки

| Тариф | Цена | Доступ |
|---|---|---|
| **Free** | $0 | Топ-3 идеи недели, без деталей |
| **Explorer** | $29/мес | Полный поток идей, финансовые оценки, лендинги |
| **Builder** | $79/мес | Ранний доступ (на 48ч раньше публичной публикации) + экспорт в PDF/Notion |
| **Investor** | $199/мес | Всё выше + возможность «зарезервировать» идею (эксклюзив на 7 дней) |

### Монетизация дополнительно

- **Marketplace реализаций**: если подписчик реализовал идею → делится % с платформой или платит за featured placement
- **Кастомный поиск**: корпоративный план — компания задаёт свою нишу, агент ищет идеи только для неё
- **Белый лейбл**: другие инкубаторы используют движок под своим брендом

## 5. Технический стек (предварительно)

### Инфраструктура

- **VPS** или домашний сервер с GPU (RTX 3090/4090 для локальных моделей)
- **OS**: Linux (Ubuntu 22.04)
- **Оркестрация агентов**: Python + asyncio или LangGraph
- **Vector Store**: ChromaDB (локально) или Qdrant (self-hosted)
- **База знаний**: Obsidian Vault (Markdown файлы, Git-синхронизация)
- **Фронтенд**: Django или FastAPI + простой React/Next.js

### AI-модели

- **Локальная модель** для фоновых циклов: Qwen 30B или DeepSeek-R2 (снижает стоимость)
- **API-модель** для финального контента и ранжирования: Claude Sonnet / GPT-4o
- **Embeddings**: `nomic-embed-text` (локально) или OpenAI `text-embedding-3-small`

### Источники данных

- `arxiv` Python library (официальный API, без лимитов)
- `huggingface_hub` (HfApi, публичные датасеты бесплатно)
- GitHub API (trending репозитории)
- Hacker News API (algolia.hn.algolia.com, бесплатно)
- RSS-ленты технологических медиа

## 6. Конкуренты и позиционирование

| Конкурент | Что делает | Разрыв |
|---|---|---|
| **Product Hunt** | Голосование за готовые продукты | Только готовые, нет AI-генерации |
| **Stratup.ai** | AI генерирует идеи по запросу | Нет автоматики, нет сообщества |
| **Y Combinator** | Инкубирует стартапы | Закрытый, нужны люди и капитал |
| **One Vote Up** | Голосование за идеи | Нет AI, нет оформления |
| **Hacker News** | Обсуждение технологий | Нет структуры, нет фокуса на идеях |

**Позиционирование:** «Product Hunt, но идеи генерирует AI, а не люди — и каждая идея уже приходит с бизнес-оберткой»

## 7. Риски

### Критические риски

1. **Качество идей** — AI может генерировать поверхностное, очевидное. Нужна жёсткая фильтрация и человеческий куратор на старте.
2. **Холодный старт сообщества** — без активных пользователей нет сигнала «что интересно». Первые 200-500 пользователей нужно привлечь вручную.
3. **Монетизация vs. openness** — если всё бесплатно, нет дохода. Если всё платно, нет сообщества.

### Операционные риски

4. **Стоимость инференса** — частые вызовы сильных LLM дорогие. Нужен баланс: локальные модели для фона, API только для финала.
5. **Авторские права на идеи** — кому принадлежит идея, которую сгенерировал AI? Нужна юридическая позиция.
6. **Актуальность** — идеи стареют. Нужен механизм архивирования и обновления.

## 8. MVP — минимально жизнеспособная версия

Что нужно сделать в первую очередь, чтобы проверить гипотезу:

1. **Агент-коллектор** (arXiv + HuggingFace) → Markdown заметки локально
2. **Synthesis Agent** (простой LLM-промпт: «найди бизнес-идею в пересечении этих концепций»)
3. **Статический сайт** с 10-20 первыми идеями (без подписки, просто для обратной связи)
4. **Telegram-канал** как первый «продукт» — публикация 1 идеи в день
5. **Метрика проверки**: 100 подписчиков Telegram-канала за первый месяц → есть спрос

**MVP не требует**: системы подписки, векторной базы, сложной оркестрации, лендингов для каждой идеи.

## 9. Вопросы, требующие ответа

Эти вопросы должны быть проработаны до начала разработки:

- Кто конкретно платит подписку? (серийный предприниматель? ангел-инвестор? студент MBA?)
- Как измерить «качество идеи» до того, как её кто-то реализовал?
- Сколько идей в неделю — оптимальное количество для подписчика?
- Как обеспечить «эксклюзивность» для платных тарифов, если AI может генерировать бесконечно?
- Как строить обратную связь — чтобы система обучалась вкусу аудитории со временем?
- Нужен ли человек-куратор на старте или можно обойтись без него?
- Какая ниша для старта? (fintech идеи? B2B SaaS? deep tech? конкретная страна/рынок?)
