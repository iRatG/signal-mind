## 1. Что хотели проверить

### Центральная гипотеза эксперимента

> **Можно ли автоматически находить торговые сигналы на российском финансовом рынке,  
> используя LLM как генератор гипотез + DuckDB как верификатор?**

Конкретные вопросы:

| Вопрос | Метод |
|--------|-------|
| Предсказывают ли новости движение рынка через 7–90 дней? | Корреляция news_daily → v_moex_sectors с лагом |

| Накапливаются ли знания между сессиями? | knowledge.md, experiments.db, forbidden_patterns.md |

## 2. Что построили — архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                    OUROBOROS AGENT LOOP                     │
│                                                             │
│  ┌─────────┐   ┌──────────┐   ┌──────────┐   ┌─────────┐  │
│  │  Topic  │──▶│ Generate │──▶│  SQL +   │──▶│Evaluate │  │
│  │  Pool   │   │ Hyp (LLM)│   │  Repair  │   │ (LLM)   │  │
│  └─────────┘   └──────────┘   └──────────┘   └─────────┘  │
│       ▲                                            │        │
│       │         next_hypothesis ◀──────────────────┘        │
│                                                             │
│  LOOP 1: SQL self-repair (max 3 попытки, 8 типов ошибок)   │
│  LOOP 2: Ouroboros (гипотеза → данные → новая гипотеза)    │
│  LOOP 3: experiments.db (накопление датасета fine-tuning)  │
└─────────────────────────────────────────────────────────────┘
```

### Три слоя памяти агента (при старте сессии)

```
principles.md     →  аналитические принципы (конституция)
knowledge.md      →  подтверждённые находки прошлых сессий
forbidden_patterns.md  →  задокументированные SQL-антипаттерны
sql_patterns.md   →  рабочие SQL-шаблоны (few-shot)
convergence_blacklist.json  →  темы, уводящие в конвергенцию
current_regime.json  →  текущий рыночный режим
```

### Пост-сессионный аудит — Revizor

После каждого марафона запускается `revizor.py`, который:
1. Обнаруживает aliasing-баги (подмена данных в SQL)
2. Находит WEAK_R паттерны (r < 0.30, повторяются ≥3 раз)
3. Проверяет REGIME_INACTIVE (сигналы, активные только при ставке < 15%)
4. **Автоматически** вносит фиксы в `forbidden_patterns.md` и `convergence_blacklist.json`

## 5. Ошибки и методологические минусы

### P0 — Критические (угрожают валидности метрики)

#### 5.1 Инструментальный aliasing (найден и исправлен)

```sql
-- БЫЛО (баг): imoex_close — это MOEX Index, не FTSE_CHINA_50!
SELECT m.imoex_close AS ftse_china_50, s.moexog_oil_gas ...
-- CORR(IMOEX, MOEXOG) = 0.88 — тавтология, оба российских индекса

-- СТАЛО (правильно):
FROM market_data WHERE instrument = 'FTSE_CHINA_50'
-- Реальный результат: r = -0.20 (честный, не раздутый)
```

**Последствие:** завысило confirmed rate с ~20% до 67.7% в Marathon 1.  
**Исправление:** `_check_aliasing()` в hypothesis.py + `FOREIGN_INSTRUMENTS` список в revizor.py.

#### 5.2 LLM оценивает собственные гипотезы (не исправлено)

Тот же DeepSeek, что генерирует гипотезу, оценивает её подтверждение.  
**Проблема:** LLM склонен подтверждать то, что сам же написал.  
**Масштаб:** влияет на ВСЕ 3 416 записей в experiments.db.  
**Решение:** детерминированный критерий на hold-out данных (Phase 10, не реализовано).

#### 5.3 Проблема множественных проверок (не исправлено)

290+ гипотез за сессию × 14 сессий = 4000+ тестов.  
При α=0.05 **200 ложноположительных** ожидаемы случайно.  
Поправки Бонферрони или FDR не применяются.

### P1 — Серьёзные

#### 5.4 Нет out-of-sample валидации

Все сигналы найдены и проверены на одних и тех же данных 2022–2025.  
Пример нестабильности USD/RUB→MOEXFN по годам:

```
2022: r = +0.47  (умеренная)
2023: r = +0.96  (очень сильная!)
2024: r = -0.47  (ИНВЕРСИЯ!)
2025: r = +0.40  (снова умеренная)
```

Сигнал меняет знак между годами — это нестационарность, не надёжный паттерн.

#### 5.5 EUR/RUB↔USD/RUB тавтология (найдена и исправлена)

```sql
-- CORR(EUR/RUB lag1, USD/RUB) = 0.97 при ЛЮБОМ лаге
-- Оба — рублёвые курсы, структурно коинтегрированы
```

**Последствие:** 65 итераций санкционной конвергенции в Marathon 3 (все score=85).  
**Исправление:** раздел в forbidden_patterns.md + permanent blacklist для fingerprint.

#### 5.6 knowledge.md — самоусиливающийся цикл

Подтверждённые (LLM) сигналы → knowledge.md → влияют на следующие гипотезы.  
Если начальная оценка ошибочна (P0.2) → знание накапливает ошибку.

### Сигнал 1: USD/RUB → MOEXFN (лаг 14 дней)

**Гипотеза:** Ослабление рубля предсказывает рост финансового сектора через 14 дней.  
**Логика:** Девальвация рубля → переоценка валютных активов банков → рост котировок MOEXFN.

**Верификация:**

```sql
WITH lagged AS (
    SELECT m1.trade_date,
           m1.usd_rub,
           s.moexfn_finance
    FROM v_market_context m1
    JOIN v_moex_sectors s ON s.trade_date = m1.trade_date + INTERVAL 14 DAYS
    WHERE m1.trade_date BETWEEN '2022-01-01' AND '2025-12-31'
      AND m1.usd_rub IS NOT NULL
      AND s.moexfn_finance IS NOT NULL
)
SELECT ROUND(CORR(usd_rub, moexfn_finance), 4) AS corr, COUNT(*) AS n
FROM lagged
```

**Результат:**

| Метрика | Значение |
|---------|---------|
| Корреляция (overall) | **r = +0.758** |
| Выборка | n = 990 |
| Интерпретация | Сильная положительная связь |

**Разбивка по годам:**

```
2022:  r = +0.47  ████████░░░░  n=232  (умеренная)
2023:  r = +0.96  ██████████░░  n=253  (очень сильная)
2024:  r = -0.47  ░░░░░░████░░  n=252  (ИНВЕРСИЯ!)
2025:  r = +0.40  ███████░░░░░  n=253  (умеренная)
```

**Вывод:**

- Общая корреляция r=0.758 красивая, но **нестабильна по годам**.
- В 2024 сигнал **инвертировался** — рост рубля сопровождался ростом MOEXFN.
- Причина нестабильности: в 2024 ЦБ поднял ставку до 16–21%, банки зарабатывали на высоких процентных доходах вне зависимости от курса рубля.
- **Статус: условный сигнал** — работает в нормальном рублёвом режиме, не работает при высоких ставках ЦБ.

### Сигнал 2: MSCI_INDIA → MOEXFN (лаг 14 дней, режим сильного рубля)

**Гипотеза:** Индийский рынок акций предсказывает российский финансовый сектор через 14 дней.  
**Логика:** Оба — развивающиеся рынки (EM). Рост аппетита к риску → приток в EM → рост MOEXFN.

**Верификация:**

```sql
WITH lagged AS (
    SELECT d.trade_date,
           d.close        AS msci_india,
           s.moexfn_finance,
           m.usd_rub,
           CASE WHEN m.usd_rub > 80 THEN 'weak_ruble'
                ELSE 'strong_ruble' END AS regime
    FROM market_data d
    JOIN v_moex_sectors s ON s.trade_date = d.trade_date + INTERVAL 14 DAYS
    JOIN v_market_context m ON m.trade_date = d.trade_date
    WHERE d.instrument = 'MSCI_INDIA'
      AND d.trade_date BETWEEN '2022-01-01' AND '2025-12-31'
      AND d.close IS NOT NULL AND s.moexfn_finance IS NOT NULL
)
SELECT regime,
       ROUND(CORR(msci_india, moexfn_finance), 4) AS corr,
       COUNT(*) AS n
FROM lagged
GROUP BY regime
ORDER BY regime
```

**Результат:**

| Режим | Корреляция | Выборка |
|-------|-----------|---------|
| USD/RUB ≤ 80 (сильный рубль) | **r = +0.772** | n = 380 |
| USD/RUB > 80 (слабый рубль) | r = +0.129 | n = 603 |
| Общая | r = +0.625 | n = 983 |

```
Сильный рубль (USD/RUB ≤ 80):  r = +0.77  ████████░░░░  n=380  ✅
Слабый рубль  (USD/RUB > 80):  r = +0.13  ██░░░░░░░░░░  n=603  ❌
```

**Вывод:**

- Сигнал **режимно-условный**: работает только при сильном рубле.
- При текущем режиме USD/RUB ≈ 85 → сигнал **неактивен** (r=0.13, не торгуемо).
- Исторически (2022–2023, сильный рубль): MSCI_INDIA был реальным опережающим индикатором MOEXFN.
- Это подчёркивает P0.4: **нужен out-of-sample тест** — сигнал может оказаться in-sample артефактом.
- **Статус: теоретически интересный, сейчас неактивен.**

### Сигнал 3: Brent → MOEXFN (лаг 90 дней)

**Гипотеза:** Нефть предсказывает финансовый сектор через 90 дней (обратная зависимость).  
**Логика:** Brent↑ → рубль укрепляется → ЦБ снижает ставку → маржа банков падает → MOEXFN↓.

**Верификация:**

```sql

           s.moexfn_finance

    JOIN v_moex_sectors s ON s.trade_date = m1.trade_date + INTERVAL 90 DAYS
    WHERE m1.trade_date BETWEEN '2022-01-01' AND '2025-12-31'
      AND m1.brent_usd IS NOT NULL AND s.moexfn_finance IS NOT NULL
)
SELECT ROUND(CORR(brent_usd, moexfn_finance), 4) AS corr, COUNT(*) AS n
FROM lagged
```

**Результат:**

| Метрика | Значение |
|---------|---------|
| Корреляция (overall) | **r = -0.702** |
| Выборка | n = 799 |
| Интерпретация | Сильная обратная связь |

**Разбивка по годам:**

```
2022:  r = -0.57  ░░████████░░  n=199  (умеренная обратная)
2023:  r = -0.11  ░░░░░░░████░  n=202  (слабая)
2024:  r = -0.10  ░░░░░░░░███░  n=199  (слабая)
2025:  r = +0.26  █████░░░░░░░  n=199  (СМЕНА ЗНАКА!)
```

**Вывод:**

- Общая r=-0.702 сильная, но **уходит к нулю и меняет знак**.
- 2022 имел реальную связь Brent→рубль→ставка→MOEXFN.
- С 2023: ЦБ поднял ставку до рекорда, банки стали зарабатывать на ставке независимо от нефти.
- 2025: знак инвертировался — нефть и банки снова движутся в одном направлении.
- **Статус: исторически реальный, сейчас слабый и нестабильный.**

## 7. Что поняли — ключевые выводы

### Технические

1. **SQL self-repair работает.** 28 из 28 починено, 0 безнадёжных. 8 типов ошибок DuckDB закрыты.

2. **Anti-convergence необходим.** Без blacklist агент застревал на одной теме (65 повторений санкций с score=85 — все тавтология).

3. **Revizor.apply_fixes() — правильная архитектура.** Feedback-петля: марафон → аудит → патч конфигов → следующий марафон лучше. One-time code, self-updating data.

4. **Режимная инъекция в генератор — ошибка.** При добавлении `current_regime` в промпт генератора LLM начал генерировать сравнительные SQL с `key_rate < 15%` → Revizor ложно их флагировал → confirmed rate упал с 17% до 5%. Правило: инъекция режима нужна только в **evaluator**, не в generator.

5. **Fingerprint normalization работает.** `re.sub(r"\d+", "N", text[:55])` обнаруживает семантические повторы при разных числах (lag=7 и lag=14 — одна тема).

### Научные

6. **Сигналы нестационарны.** Все три топ-сигнала меняют коэффициент или знак по годам. Суммарная корреляция 2022–2025 может быть артефактом конкретного периода.

7. **Режимные разрывы — реальные и важные.** Финансовый рынок РФ имеет чёткие режимы (ставка 7% vs 21%, рубль 50 vs 135). Внутри режима сигналы сильнее. Это полезная находка даже без OOS.

8. **LLM не может независимо верифицировать свои гипотезы.** r=0.758 и r=0.98 — LLM не видит разницы. Детерминированный порог на SQL-результате + hold-out данные — единственный путь.

9. **knowledge.md — двусечущий меч.** Ускоряет нахождение новых гипотез, но если содержит ложные сигналы — усиливает их. Нужна проверка перед записью (автоматическая SQL-верификация, не LLM-оценка).

10. **$10 за 3416 гипотез — это дёшево.** DeepSeek API даёт возможность масштабировать EDA за разумные деньги. Основная ценность — не "торговые сигналы", а накопленный Text2SQL датасет.
